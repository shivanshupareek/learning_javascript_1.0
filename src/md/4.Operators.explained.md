### JavaScript Operators — Explained

This document explains the operators demonstrated in `src/content/4.Operators.js`. It covers arithmetic, comparison, logical, assignment, unary, and ternary operators, with short examples and gotchas.

---

#### Arithmetic Operators
- `+` Add or concatenate
  ```js
  10 + 5; // 15
  'Hello ' + 'World'; // 'Hello World'
  ```
- `-` Subtract
  ```js
  10 - 5; // 5
  ```
- `*` Multiply
  ```js
  10 * 5; // 50
  ```
- `/` Divide
  ```js
  10 / 5; // 2
  ```
- `%` Modulus (remainder)
  ```js
  10 % 5; // 0 (10 is divisible by 5)
  10 % 4; // 2 (10 = 8 + 2)
  10 % 3; // 1 (10 = 9 + 1)
  ```
- `**` Exponentiation
  ```js
  10 ** 5; // 100000
  ```

---

#### Comparison Operators
- `>` Greater than
- `<` Less than
- `>=` Greater than or equal to
- `<=` Less than or equal to
- `==` Loose equality (value only; allows type coercion)
- `===` Strict equality (value and type; preferred)
- `!=` Loose inequality
- `!==` Strict inequality

Examples:
```js
5 > 3; // true
'5' == 5;  // true (coerces string to number)
'5' === 5; // false (different types)
```

---

#### Logical Operators
- `&&` AND — true if both operands are truthy
- `||` OR — true if at least one operand is truthy
- `!` NOT — negates truthiness

Short-circuiting behavior (also returns the last evaluated operand):
```js
true && 'OK';   // 'OK'
false && 'OK';  // false
false || 'ALT'; // 'ALT'
'val' || 'ALT'; // 'val'
!true;          // false
```

From the code:
```js
const a = 20, b = 50;
if (a > 10 && b < 100) {
  console.log('Yes, both the conditions are true');
}

const isAdmin = true, isLoggedIn = false;
if (isAdmin || isLoggedIn) {
  console.log('Access Granted!');
}

const temp = 30;
if (!(temp < 30)) {
  console.log("Yes it's hot");
}
```

---

#### Assignment Operators
- `=` Assign
- `+=` Add and assign
- `-=` Subtract and assign
- `*=` Multiply and assign
- `/=` Divide and assign
- `%=` Remainder and assign
- `**=` Exponentiate and assign

Examples:
```js
let x = 10;    // assignment
x += 5;        // x = x + 5  -> 15
x -= 3;        // x = x - 3  -> 12
x *= 2;        // x = x * 2  -> 24
x /= 6;        // x = x / 6  -> 4
x %= 3;        // x = x % 3  -> 1
x **= 4;       // x = x ** 4 -> 1
```

---

#### Unary Operators
- `+` Unary plus — converts to number
  ```js
  +'42'; // 42
  +true; // 1
  +null; // 0
  ```
- `-` Unary minus — numeric negation
  ```js
  -'3'; // -3
  ```
- `typeof` Type inquiry
  ```js
  typeof 123;     // 'number'
  typeof 'text';  // 'string'
  typeof null;    // 'object' (quirk)
  typeof [];      // 'object'
  typeof undefined; // 'undefined'
  ```
- `++` Increment; `--` Decrement
  - Pre-increment (`++a`) updates first, then yields the new value
  - Post-increment (`a++`) yields the old value, then updates
  ```js
  let a = 1;
  ++a;      // 2 (a is now 2)

  let b = 1;
  const c = b++ + b; // (1) + (2) = 3; b becomes 2 after expression

  let d = 10;
  let e = d++;      // e = 10, d = 11

  let f = 10;
  let g = ++f;      // g = 11, f = 11

  let h = 10;
  const result = h++ + ++h; // 10 + 12 = 22; h becomes 12
  ```
- `!` Logical NOT
  ```js
  const cVal = null;
  !!cVal; // false (double NOT to coerce to boolean)
  !cVal;  // true
  ```

Truthiness example:
```js
let c = null;
if (c) {
  console.log("it's truthy");
} else {
  console.log("it's falsy");
}
!c ? console.log("it's truthy") : console.log("it's falsy");
```

#### Ternary Operator
- Syntax: `condition ? exprIfTrue : exprIfFalse`
- Nested ternaries can express multi-branch decisions but can reduce readability.

Example from the code:
```js
let score = 45;
let result = score >= 90 ? 'Incredible' :
             score >= 80 ? 'Amazing' :
             score >= 70 ? 'Great' :
             score >= 60 ? 'Not bad' :
             score >= 50 ? 'can do better' :
             score >= 40 ? 'Super close' :
             score >= 30 ? 'Barely passed' :
             'fail';
console.log(result); // 'can do better' for 50, 'fail' for < 30, etc.
```

Tip: For complex branching, consider `if/else if/else` for clarity.

---

#### Loop + Increment Gotcha
Post-increment inside loops can be tricky:
```js
let count = 0;
for (let i = 0; i < 5; i++) {
  count += i++;
}
console.log(count);
```
- `i++` inside the loop body increments `i` again, effectively skipping numbers.
- Be mindful of readability and intent; avoid modifying the loop variable inside the body unless necessary.

---

#### Notes & Best Practices
- Prefer `===`/`!==` over `==`/`!=` to avoid implicit type coercion issues.
- Use parentheses to make precedence explicit when mixing operators.
- Avoid deeply nested ternaries; they can be hard to read and maintain.
- Remember that `&&` and `||` return operands, not booleans, which is useful for default values or conditional execution.
